## Key Insights

### Root Cause Resolution

**Yesterday's problem:** Model IoU stuck at 0.0-0.06, unable to cross threshold.

**Today's solution:** Expanding proximity zones from 10px to 20px fundamentally changed the learning landscape:
- **Larger target area**: More pixels with positive values (1-20 vs 1-10)
- **Better class balance**: Reduced background dominance
- **Easier threshold crossing**: With values up to 20, crossing 1.0 is more achievable
- **Stronger learning signal**: Model receives more positive examples per tile

### Why 20px Works

1. **Spatial Context**: 20px provides more spatial context around lobes
2. **Gradient Information**: Gradual decay (20 → 1) provides rich distance information
3. **Training Signal**: More pixels contribute to loss, reducing background dominance
4. **Threshold Margin**: Values up to 20 give model margin to learn threshold crossing

### Model Performance Analysis

**IoU Progress:**
- 10px: 0.0647 (stuck)
- 20px: 0.1402 (learning) ✅

**Relative Performance:**
- 10px: Model 7.51x worse than baseline
- 20px: Model 4.05x worse than baseline (improved, but still needs work)

**Status:** Model is now learning, but still has room for improvement. IoU of 0.14 is meaningful progress but far from production-ready.

## Critical Analysis: Is the Improvement Real?

**Important Question:** With 4x larger target area (20px vs 10px), is the IoU improvement due to better learning or just easier targets?

**Random Baseline Analysis:**
- **20px random IoU** (threshold=5.0): ~0.054 (5.4%)
- **10px estimated random IoU**: ~0.021 (2.1%)
- **Ratio**: 2.54x easier with 20px

**Model Performance vs Random:**
- **10px model**: IoU = 0.0647 → **3.1x better than random** (0.0647 / 0.021)
- **20px model**: IoU = 0.1402 → **2.6x better than random** (0.1402 / 0.054)

**Key Insight:** The model is actually performing **relatively worse** on 20px compared to random baseline (2.6x vs 3.1x), even though absolute IoU is higher. This suggests:
1. The task is indeed easier with 20px (larger target area)
2. The model may not be learning proportionally better
3. The improvement might be partially due to easier targets, not just better learning

**However**, the absolute IoU improvement (0.0647 → 0.1402) is still meaningful because:
- Model is now crossing threshold consistently (was stuck before)
- Detection capability exists (IoU > 0.1 indicates real detection)
- The relative performance gap (model vs baseline MAE) improved (7.51x → 4.05x)

**Conclusion:** The 20px expansion made the task easier, which helped the model learn, but the model's relative performance vs random baseline didn't improve proportionally. The breakthrough is real (model is detecting), but we need to push further to achieve truly superior learning.
