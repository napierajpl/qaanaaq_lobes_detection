## Segmentation layer

Work on an **optional segmentation layer** as a hint for the CNN (e.g. boundary cues):

- **Script:** `scripts/create_segmentation_layer.py` — builds a separate raster of segment IDs (Felzenszwalb, OBIA-style) from the input imagery. Optional two scales; block processing for large rasters.
- **Boundary:** By default limited to the research boundary (`-b data/raw/vector/research_boundary.shp`); pixels outside are nodata. Omit `-b` to segment the full raster.
- **Use in training:** Output can be tiled (same size/overlap as feature tiles) and added as a **6th input channel** via `data.use_segmentation_layer: true` and `paths.*.segmentation_dir` in config. Does not change the target.
- **Dependency:** `pip install scikit-image` (not in Poetry to avoid conflicts). See README “Segmentation layer (optional)” and `docs/synthetic_parenthesis_dataset.md`.

**Potential meaning for training:** The segmentation channel provides pre-computed region boundaries (object-like segments). The hypothesis is that lobe edges often align with such boundaries (terrain, texture, or spectral homogeneity). Feeding segment IDs as an extra input can (1) give the model a direct cue for where boundaries might lie, (2) reduce the need to learn low-level edges from RGB alone, and (3) potentially improve localization and IoU, especially where lobes are subtle. The model can learn to use or downweight this channel; the training target (proximity or binary) is unchanged, so the segmentation layer is an optional auxiliary input to be validated in experiments.
Segment boundaries can be also used for shaping predictions.

## Change of synthetic parenthesis generation

Synthetic parenthesis dataset can be generated in two ways; we added and use the **full-raster** flow:

- **From full raster (recommended):** `scripts/generate_synthetic_parenthesis_from_raster.py` — loads full RGB, uses research boundary (and optional valid-data polygons) so shapes are placed **only inside** the boundary and avoid white areas. Draws parenthesis (e.g. 2000 shapes, configurable height), builds target raster (0/20), stacks RGB+DEM+slope, then tiles to **256 and 512** in one run. Output: `data/processed/tiles/synthetic_parenthesis_256/` and `synthetic_parenthesis_512/` (features, targets, `filtered_tiles.json`). Uses `src/data_processing/boundary_extraction` and `src/data_processing/synthetic_shapes`.
- **Legacy (tile-based):** `scripts/generate_synthetic_parenthesis_dataset.py` — generates from existing tiles (one tile size per run). Still available for compatibility.
- **Docs:** `docs/synthetic_parenthesis_dataset.md` describes both flows; README “Synthetic parenthesis dataset” points to the full-raster script and training with `--config configs/training_config_synthetic_parenthesis.yaml --mode synthetic_parenthesis`.
