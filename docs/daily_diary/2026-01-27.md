## What We Added

### 1. Optuna tuning workflow (new scripts + docs)

- **`scripts/tune_hyperparameters.py`**: runs an Optuna study and launches multiple trials.
- **`scripts/train_model.py`**: refactored to expose `train_model_with_config(config, mode, trial, run_name)` so Optuna can call training directly.
- **`START_HP_TUNING.sh`**: convenience wrapper for launching a ~8-hour tuning session.
- **`docs/OPTUNA_QUICK_START.md`**: “how to run” guide for tuning.

### 2. Tuning search space (initial “all-nighter” scope)

Per our decision, the study tunes:

- **Core (start with 5):**
  - `training.learning_rate` (loguniform \(1e-4\)–\(1e-1\))
  - `training.batch_size` (categorical: 8/16/32/64)
  - `training.weight_decay` (loguniform \(1e-5\)–\(1e-2\))
  - `training.focal_alpha` (uniform 0.25–0.95)
  - `training.focal_gamma` (uniform 1.0–4.0)
- **Extra for the all-nighter:**
  - `training.loss_function` (categorical: focal / combined / weighted_smooth_l1)
  - `model.encoder.name` (categorical: resnet50 / resnet152 / swin_v2_base / swin_v2_tiny)

Trial-level MLflow experiment name is set to **`lobe_detection_hp_tuning`**.

### 4. Tuning session “resume/seed” without a database (CSV history)

We implemented seeding from previous best hyperparameters and made it visible in the terminal:
- `[RESULTS] csv_path=...` prints where tuning reads/writes its history
- `[SEED] strategy=... params=...` prints whether (and what) we seed from

We store tuning history as a **single append-only CSV** under:
- `data/optuna_results/lobe_detection_hp_tuning_trials.csv`

### 5. Training log readability improvements

To make long runs easier to monitor:
- Per-epoch one-line summary including `best_val_loss`, early-stop counter, and baseline improvement
- `[BEST] ...` line when a new best is achieved
- Reduced noisy warnings in logs (PyTorch/ReduceLROnPlateau)

### 3. Dependency install workaround (Poetry lock would downgrade torch)

Attempting `poetry install` after adding Optuna triggered Poetry to try downgrading **PyTorch** (to 2.1.2), which fails on this Windows environment.

Resolution: install Optuna **manually** inside the Poetry venv:

- `poetry run pip install optuna`

Documentation added:
- **`INSTALL_OPTUNA.md`**


Expected outputs:

- Best params saved to `configs/best_hyperparameters.yaml`
- Trial runs logged under MLflow experiment `lobe_detection_hp_tuning`

## Notes

- Running multiple shorter tuning sessions (e.g., 10 trials + 10 trials) is operationally safer than a single very long run, but it slightly reduces sampler continuity when using in-memory studies.
