## Key changes

### 1. HP tuning progress plots (two files)

- **`data/optuna_results/<study_name>_progress.png`**  
  Trial number vs final value (and best-so-far). Updated after each trial. Same as before; path printed at start.

- **`data/optuna_results/<study_name>_progress_epochs.png`** (new)  
  **All epochs, all trials** on one plot:
  - X-axis: labels like **T0 E1**, **T0 E2**, …, **T1 E1**, … (trial index + epoch).
  - Y-axis: validation loss at that epoch.
  - Data from MLflow: for each completed trial we use `mlflow_run_id` from user_attrs and fetch `val_loss` history, then concatenate in trial order.
  - Updated after each trial so you can watch tuning live.

Epochs per trial come from config **`training.num_epochs`** (e.g. 300); a trial can stop earlier due to pruning or early stopping.

### 2. Training: loss plot for Optuna trials and live file update

- **Placeholder:** Initial `plots/loss.png` is now created for **every** run (including Optuna trials), so the path printed at start exists immediately.
- **Per-epoch update:** After each epoch we call `create_training_plots()` and log the loss figure; for **Optuna trials** we now also update the loss plot (we only log the loss figure, not the other plots, to keep I/O light).
- **Live file:** When tracking is `file:`, we **write the loss figure to the artifact path** on disk after each epoch (`fig.savefig(loss_plot_path)`), so the printed path (e.g. `./mlruns/.../artifacts/plots/loss.png`) updates every epoch and can be refreshed in a viewer.

### 3. Visualization: single-epoch xlim warnings removed

- **Issue:** With one epoch, `ax.set_xlim(left=1, right=1)` triggered matplotlib “identical low and high xlims” (singular transformation).
- **Change:** In `src/training/visualization.py`, added **`_xlim_epochs(epochs)`**: when `min(epochs) == max(epochs)` we use `(min - 0.5, max + 0.5)`. Used in `plot_mae_comparison`, `plot_loss`, and `plot_iou`. Warnings no longer appear on first epoch.

### 4. README and testing

- **README:** Epochs per trial and the two tuning progress plots documented; cross-reference for `--hp_from_run_id` under Hyperparameter tuning; Testing section (unit + e2e, `pytest tests/unit`, `pytest tests/e2e -m e2e`).
- **E2E tests:** `tests/e2e/test_e2e_pipeline.py` – minimal training (1 epoch, dev 256), minimal tuning (1 trial), optional data-prep-then-training; require dev data; run with `pytest tests/e2e -m e2e`.

## Lessons learned

- Tuning progress is easier to follow with both a trial-level summary and an all-epochs plot; MLflow `get_metric_history(run_id, "val_loss")` gives per-epoch data for the epochs plot.
- Writing the loss figure to the artifact path on disk (in addition to `mlflow.log_figure`) makes the printed path actually update each epoch when using `file:` tracking.
- Single-point x axes need a small range (e.g. ±0.5) to avoid matplotlib singular-transform warnings.
