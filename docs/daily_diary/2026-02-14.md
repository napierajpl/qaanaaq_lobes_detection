# Daily Diary - February 14, 2026

## Overview

Added **train subsampling** (option 3 from “training forever” mitigations): config `train_subsample_ratio: 0.3` and per-epoch random subset of training tiles. Reduces time per epoch (e.g. ~30% of batches) and avoids overtraining by seeing a different random 30% each epoch. E2E tests run; changes pushed.

---

## Key changes

### 1. Training config: subsampling option

- **`configs/training_config.yaml`** → under `data`:
  - **`train_subsample_ratio: 0.3`** — use 30% of training tiles per epoch.
  - Comment: value in (0, 1] (1.0 = no subsampling); different random subset each epoch.

### 2. Training script: per-epoch random subset

- **`scripts/train_model.py`**:
  - Read **`config["data"].get("train_subsample_ratio", 1.0)`**. If `< 1.0`, log subsampling and set up per-epoch subset.
  - **Each epoch** (when subsampling): `np.random.default_rng(epoch)` → sample `max(1, int(len(train_tiles) * ratio))` indices without replacement → build `epoch_tiles` → `create_dataloaders(epoch_tiles, val_tiles, ...)` → use that `train_loader` for `train_one_epoch`.
  - **MLflow:** log `data.train_subsample_ratio` when subsampling is used.
  - Full `train_tiles` list unchanged; only the loader is built from a subset each epoch.

Effect: with 173 batches per epoch at 2 min/batch, 30% subsampling gives ~52 batches per epoch and much faster iteration while still varying the training set every epoch.

### 3. E2E: minimal config without extended set

- **`tests/e2e/conftest.py`**: In `make_minimal_config`, set **`use_background_and_augmentation: False`** so the minimal training test uses only `filtered_tiles.json` and does not require `extended_training_tiles.json` (which is not created by the default dev data prep).
