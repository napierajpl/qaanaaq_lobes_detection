## Throttle mechanism change (save best model)

**What we changed:** During training, the “save best model” throttle was **epoch-based** (only save if at least 10 epochs since last save). We switched it to **time-based**: save only if at least **3 minutes** have passed since the last save.

**Rationale:** With variable epoch length, 10 epochs can be much longer or shorter than desired. A 3-minute throttle limits disk/MLflow churn while still allowing periodic checkpoints when the model improves frequently.

**Implementation:** In `scripts/train_model.py`:

- Replaced `last_epoch_saved` and `SAVE_THROTTLE_EPOCHS = 10` with `last_time_saved` and `SAVE_THROTTLE_SECONDS = 3 * 60`.
- `may_save` now uses `time.time() - last_time_saved >= SAVE_THROTTLE_SECONDS` (or first save).
- After saving we set `last_time_saved = time.time()`.

No new user-facing config or CLI; behavior is internal to training.
