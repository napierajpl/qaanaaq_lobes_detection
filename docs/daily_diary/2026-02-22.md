# Daily Diary - Sunday 22 February 2026

## Slope-stripes channel (Gabor): process and proof

We added a **slope-stripes** optional input channel: a raster that is high only where linear texture in the imagery is **aligned with terrain slope** (aspect). It is built from RGB + DEM using Gabor filters (freq=0.15, sigma=5.0).

**Process (high level):**

1. **Texture strength** — Gabor filters at multiple orientations; strength = max response (normalized).
2. **Stripe direction** — orientation of the filter that gave the max response (+ 90°).
3. **Slope alignment** — compare stripe direction to aspect (steepest descent from DEM); alignment in [0, 1].
4. **Channel** — strength × alignment, clipped to [0, 1]. High only where stripes exist and follow slope.

**Proof that alignment detection works:**

- **Aligned case:** On normal (unrotated) imagery, slope-aligned stripes produce strong values.

  ![Gabor slope-stripes sample (freq=0.15, sigma=5.0)](../../data/processed/raster/dev/gabor_slope_stripes_sample_freq0_15sigma5_0.png)

- **Misaligned case:** When RGB is rotated 90° but DEM is unchanged, stripe direction no longer matches slope; the channel drops (sanity check).

  ![Gabor rot90 sanity (expect low values)](../../data/processed/raster/dev/gabor_rot90_sanity.png)

**Implementation:** Script `scripts/create_slope_stripes_channel.py` with `--method gabor --gabor-frequency 0.15 --gabor-sigma 5.0`. Default output: `data/processed/raster/slope_stripes_channel.tif`. Tile with `create_tiles.py` (same size/overlap as features); enable in config with `data.use_slope_stripes_channel: true` and `slope_stripes_channel_dir`. Full pipeline (`prepare_training_data.py`) also generates and tiles this channel for the normal lobe dataset. Synthetic parenthesis dataset to be wired later.
